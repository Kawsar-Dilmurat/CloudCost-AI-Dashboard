1. Fix Cloud Health Score logic
The Recommendations page currently shows an Optimization Score of 0, which looks too harsh and may look like a bug.

Rename:
- “Optimization Score” → “Cloud Health Score”

Use a more realistic score calculation:
- Start from 100
- Subtract 5 points for each critical issue
- Subtract 4 points for each high priority issue
- Subtract 2 points for each medium priority issue
- Subtract 1 point for each low priority issue
- Subtract an additional penalty based on savings percentage:
  - savings percentage < 10%: subtract 5
  - 10%–25%: subtract 10
  - 25%–40%: subtract 15
  - over 40%: subtract 20
- Clamp score between 15 and 100, unless there are no resources loaded.

For the current demo data, the score should not be 0. It should probably be around 25–45 and show status “Critical” or “Needs Attention”.

Status labels:
- 80–100: Healthy
- 60–79: Good
- 40–59: Needs Attention
- 15–39: Critical

7. Resource Detail View
Add a resource detail modal or side panel on the Resources page.

Each resource row should have a “View Details” button or be clickable.
When opened, the detail view should show:
- Resource name
- Resource type
- Region
- Environment
- Monthly cost
- Usage/status
- Issue detected
- Priority
- Estimated monthly savings
- Why this resource was flagged
- Recommended action
- Risk note before applying the action

Use resource-type-specific explanations:

For underutilized EC2:
Why flagged: “Average CPU usage is below the 10% threshold.”
Recommended action: “Right-size the instance, stop it during off-hours, or validate whether it can be terminated.”
Risk note: “For production workloads, verify traffic patterns and dependencies before making changes.”

For oversized RDS:
Why flagged: “CPU utilization is below 15% while monthly cost is above $100.”
Recommended action: “Review instance class and consider downsizing during a maintenance window.”
Risk note: “Validate performance requirements, backups, and maintenance windows before resizing.”

For unattached EBS:
Why flagged: “The volume is not attached to any EC2 instance but is still generating monthly storage cost.”
Recommended action: “Create a snapshot if needed, then delete the unattached volume.”
Risk note: “Confirm the volume is not needed for rollback or recovery before deletion.”

For missing S3 lifecycle policy:
Why flagged: “The bucket has no lifecycle policy and may be storing old objects in expensive storage classes.”
Recommended action: “Add lifecycle rules to transition older objects to S3 Standard-IA or Glacier.”
Risk note: “Confirm retention and compliance requirements before archival or deletion.”

For security issue:
Why flagged: “Port 22 is open to the public internet.”
Recommended action: “Restrict SSH access to trusted IP ranges or use AWS Systems Manager Session Manager.”
Risk note: “Changing access rules may impact administrators who currently rely on SSH.”

For reliability issue:
Why flagged: “Production resource does not have CloudWatch alarms configured.”
Recommended action: “Add CloudWatch alarms for CPU, memory if available, storage, and error rates.”
Risk note: “Alarms should be tuned to avoid alert fatigue.”
